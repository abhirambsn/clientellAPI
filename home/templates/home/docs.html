Documentations!

The 3 tables, namely `Accounts`, `Users` and `Opportunity` have been fetched from salesforce using django-salesforce library. These tables are then stored to models.py using command python manage.py inspectdb --database=salesforce > models.py. Then these models were used to create APIViews using django-rest framework and were given urls in `/api/table-name/page-number` syntax. Salesforce API integration requires 5 tokens, namely `Host`, `Cliend ID`, `Client Secret`, `Username` and `Password`. For development purposes, these keys are hard coded to settings.py but upon deployment, they are served as environment variables to the server. As the django-salesforce loads data from Salesforce database itself, there was no need to develop API endpoint `import`, as every time the API is accessed, django automatically re-imports all the data from the database. Thus the data is never saved locally, which reduces memory requirements.

Access the links from the side navigation bar to view working of the APIs and the logic and idea behind them!
